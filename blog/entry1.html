Last updated: @Feb 21, 2021

In this project, we built a sequence to sequence model to map to actions for an agent in the ALCHEMY environment from the SCONE dataset. The code for the project can be found at https://github.com/carriex/scone.

Model

The model is a sequence to sequence model consisting of two encoders and one decoder. For all the sequences (instruction, states, actions), paddings and an end of sentence token are added accordingly.

Instruction encoder

The instruction encoder encodes the instructions up to the current one in the interactions. Suppose the current instruction is the i i i﻿th instruction in the interaction, we encode < x ˉ 1 , x ˉ 2 , . . . x ˉ i > <\bar{x}_{1}, \bar{x}_{2}, ... \bar{x}_{i}> <xˉ1​,xˉ2​,...xˉi​>﻿ and separate each instruction with a special token SEP_PREV, and separate x ˉ i − 1 , x ˉ i \bar{x}_{i-1}, \bar{x}_{i} xˉi−1​,xˉi​﻿ with SEP_CURR. Each token is passed to an embedding layer with dimension size of 100 and then passed to a 1 layer bi-directional LSTM with hidden size of 100.

For example, an instruction will be first tokenized and padded, and then transformed into an embedding.

['pour', 'out', 'left', 'most', 'yellow', 'chemical', 'SEP_PREV', 'pour', 'yellow', 'chemical', 'into', 'left', 'most', 'orange', 'chemical', 'SEP_PREV', 'pour', 'out', 'one', 'unit', 'of', 'purple', 'chemical', 'SEP_CURR', 'mix', 'beaker', 'of', 'yellow', 'and', 'orange', 'chemicals', 'so', 'that', 'they', 'form', 'a', 'brown', 'chemical', '_EOS', '_PAD', ...,'_PAD']

State encoder

The state encoder encodes the world states S i S_{i} Si​﻿ in the form of s 1 , s 2 , . . . s 7 {s_{1}, s_{2}, ...s_{7}} s1​,s2​,...s7​﻿. Each s j s_{j} sj​﻿ is passed to an embedding layer with dimension size of 100 and then passed to a 1 layer LSTM with hidden size of 100. We encode both the initial world state S 0 S_{0} S0​﻿ and the current world state S k S_{k} Sk​﻿.

For example, s 1 s_{1} s1​﻿will be first tokenized and padded, then transformed into an embedding.

['2',':','o','y', '_EOS', '_PAD', ..., '_PAD']

Action decoder

For the decoder, the input sequence is the action word of the corresponding actions. For example, the action 'push 1 b' will be tokenized into ['push', '1', b']. To maintain the structure of the output action, one '_NONE' token is added to each POP action, and two '_NONE' tokens are added to the '_EOS' token. For example:

Actions: ['pop 6', 'pop 6', 'push 7 p', 'push 7 p']
Action words: ['pop', '6', '_NONE', 'pop', '6', '_NONE', 'push', '7', 'p', 'push', '7', 'p', '_EOS', '_NONE', '_NONE', '_PAD', '_PAD', '_PAD', '_PAD', '_PAD', '_PAD', '_PAD']

A <SOS> token is used as the initial input. we use the last hidden state and cell state from the instruction encoder as the initial hidden state. We use the hidden state to attend on hidden states of each step of the encoder outputs; the encoded current world state (hidden state) and the encoded initial world state (hidden state) for the interaction. (Note: world state is not updated during the action decoding for a single instruction). All attention layers are bilinear attention. We concat all the attended output and use a fully connected layer with tanh activation to project the concatenated output to the dimension of the hidden states. We then past the hidden state and the input to the decoder. The decoder model is a 20 layer LSTM with a hidden size of 100. The decoder LSTM has a drop out rate of 0.2.

Training

The model is optimized with negative log likelihood loss using the Adam optimizer with an initial learning rate of 0.001. A learning rate scheduler is used to reduce learning rate when validation loss stops improving. When decoding the sequence, teacher forcing is used with a ratio of 1 (i.e. we input the ground truth action y i − 1 ′ y_{i-1}^{'} yi−1′​﻿when decoding y i y_{i} yi​﻿). The model is trained for 100 epochs and we use the model with the minimum validation loss as the final model. The model is trained for 100 epoches using a batch size of 16.

Inference Procedure

The task is evaluated using two metrics: single-sentence task completion and interaction task completion. Hence there are two inference procedures to produce the corresponding results. For each given instruction, we use the instruction encoder to encode instructions and state encoder to encode current world state and initial world states. We then evaluate the accuracy by the final world state of the instruction. For each given interaction, we encode the instruction and world state in a similar fashion. However, the initial world state of the intermediate instruction is output by the previous instruction prediction. We evaluate the final world state of the interaction.

Unknown word handling

For unknown word in the instruction, we encode them into a single token UNK. For state encoding, since it is composed of colors and beaker id, there are no issue with unknown word.

Results

The model achieves an instruction level accuracy of 74% and an interaction level accuracy of 27%.

Quantitative analysis

Instruction level accuracy

First we want to understand how the model is predicted for instructions at different positions in the interaction. We can see that the error rate of the first instruction is significantly lower than the later ones in the interaction, suggesting that the model is weak in dealing with long sequences. Another reason is that the latter instructions are more context dependent. For instance, for the below instruction of the fourth instruction in the interaction, the model will need to resolve the reference for "this chemical", which refers to the "first remaining purple chemical" in the previous instruction. Moreover, it will need to correctly identify the current state of the beaker to know how many colored chemical to POP.

Instruction: 'throw out the last green chemical SEP_PREV same for the red chemical SEP_PREV same for the first purple chemical SEP_PREV same for two units of the first remaining purple chemical SEP_CURR throw out the rest of this chemical'

We can also see that the error rate is higher for instructions that are mapped to longer action sequences, as well as instruction with more words.

Action legality

We also checked whether the predicted actions are semantically legal, e.g. for a PUSH action, there must be one following token produced for the beaker, followed one token for the colored chemical. (note: this is different from whether the predicted action is valid for the world state).

Out of all predicted actions, 15% are illegal actions, among which 52% are pop actions and the remaining are push actions.

Discussion

Resolving references to state and previous instructions

As mentioned in the section above, one of the difficulty for the model is to resolve references in the instruction, especially multi-hop references (e.g. when the second instruction is dependent on the first one, and the third one is dependent on the second). Another difficulty is to correctly identify the state of the current beaker. Potential improvements can be done in both the encoders and decoders. Specifically, currently the decoder is doing all the heavy lifting whereas encoder only gets the whole instruction. One direction is to leverage more complicated model architecture (such as Transformer) to better encode those information in the encoder.

Debiasing POP and PUSH action

As discussed in the data section, the training and testing data is highly biased toward POP action. Even though we didn't perform a quantitative analysis on the error rate of PUSH and POP action (given that the actions in one instruction is orderless), this might bias the model towards always outputting POP, as discussed in this paper. A learning algorithm is also proposed to address this issue, which is also worth examining.
